# v3.5 to 3.6

## Breaking Changes

### Default local include paths now include `charts/**`

When using `argocd app diff --local --server-side-generate`, the implicit default for `--local-include` has changed from `["*.yaml", "*.yml", "*.json"]` to `["*.yaml", "*.yml", "*.json", "charts/**"]`.

This means that any files under a local `charts/` subdirectory will now be sent to the repo-server for server-side manifest generation even if they do not match the `*.yaml`, `*.yml`, or `*.json` globs. Examples include Helm helper templates (e.g., `*.tpl`), shell scripts, or other supporting assets that live under `charts/`.

> [!WARNING]
> If your application source contains non-YAML content under `charts/` that you do not want transmitted, you must override the include set via repeated `--local-include` flags to narrowly specify what should be sent.

#### Impact

- Larger payloads may be uploaded during `--server-side-generate` operations when a `charts/` directory is present.
- Repositories containing sensitive or binary files under `charts/` may unintentionally transmit those files unless the include patterns are restricted.

#### Mitigations and examples

- Restore pre-3.5 behavior (YAML/JSON only):

  ```bash
  argocd app diff my-app \
    --local ./path \
    --server-side-generate \
    --local-include "*.yaml" --local-include "*.yml" --local-include "*.json"
  ```

- Include only Helm chart YAML while excluding other assets by inclusion:

  ```bash
  argocd app diff my-app \
    --local ./path \
    --server-side-generate \
    --local-include "*.yaml" --local-include "*.yml" --local-include "*.json" \
    --local-include "charts/**/*.yaml" --local-include "charts/**/*.yml"
  ```

Argo CD matches path-aware patterns (such as `charts/**`) against the file's relative path, allowing precise control over which subtrees are included when generating the archive sent to the server.
