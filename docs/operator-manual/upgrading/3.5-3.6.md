# v3.5 to v3.6

## Breaking Changes

### Built-in Go health checks removed from gitops-engine

Built-in resource health checks that were previously implemented in Go inside the
`gitops-engine` module have been replaced with Lua scripts under
[`resource_customizations/`](https://github.com/argoproj/argo-cd/tree/master/resource_customizations).
The following Go functions have been removed from
[`gitops-engine/pkg/health`](https://github.com/argoproj/argo-cd/tree/master/gitops-engine/pkg/health):

* `GetHealthCheckFunc`
* Per-kind health check functions such as `getDeploymentHealth`, `getPodHealth`, `getHPAHealth`, etc.

`GetResourceHealth` now relies solely on the `HealthOverride` interface (Lua scripts loaded via
`util/lua` resource customizations). If your code imports the `gitops-engine` health package directly
and depended on the built-in Go checks, you must supply your own `HealthOverride` implementation to
retain health assessment. Argo CD wires this automatically via embedded Lua scripts, so end-user
Application health behavior is unchanged.

Affected built-in resource kinds (now served by embedded Lua scripts):

* `PersistentVolumeClaim`
* `Pod`
* `Service`
* `apiregistration.k8s.io/APIService`
* `apps/DaemonSet`
* `apps/Deployment`
* `apps/ReplicaSet`
* `apps/StatefulSet`
* `argoproj.io/Workflow`
* `autoscaling/HorizontalPodAutoscaler`
* `batch/Job`
* `extensions/Ingress`
* `networking.k8s.io/Ingress`
