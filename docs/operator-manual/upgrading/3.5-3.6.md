# v3.5 to 3.6

## Custom health checks and terminating resources

Argo CD no longer short-circuits Lua health checks when a resource has `metadata.deletionTimestamp` set.
Instead, each health check is responsible for reporting deletion progress explicitly.

### What changed

Previously, when `metadata.deletionTimestamp` was set (and the resource did not have the Argo CD hook finalizer
`argocd.argoproj.io/hook-finalizer`), Argo CD returned a hard-coded `Progressing` status with message
`Pending deletion` before running any custom Lua health check.

Starting in v3.6:

* Custom Lua health checks run for terminating resources.
* Bundled Lua health checks in Argo CD include an explicit deletion branch that preserves the prior
  `Progressing` / `Pending deletion` behavior (listing remaining finalizers when present).
* Built-in Go health checks for standard Kubernetes types include the same deletion handling.
* Resources with **no** health check still report `Progressing` / `Pending deletion` via a default fallback.

### Action required for custom health checks

If you maintain custom Lua health checks in the `argocd-cm` ConfigMap, review them for terminating resources.
Add an explicit deletion branch near the top of the script if you want deletion-specific messaging (for example,
mapping known finalizers to clearer explanations):

```lua
if obj.metadata ~= nil and obj.metadata.deletionTimestamp ~= nil then
  local hs = {}
  hs.status = "Progressing"
  hs.message = "Pending deletion"
  if obj.metadata.finalizers ~= nil and #obj.metadata.finalizers > 0 then
    hs.message = "Pending deletion; blocked by finalizers: " .. table.concat(obj.metadata.finalizers, ", ")
  end
  return hs
end
```

You can customize this block further—for example, to explain what each finalizer does and which controller
clears it.

Without such a branch, a health check that evaluates `status` fields may report `Healthy` or `Degraded` while
the resource is still terminating.

### Simulating before and after the upgrade

To compare behavior, copy a live resource manifest, add `metadata.deletionTimestamp` (and optional
`metadata.finalizers`), and evaluate health with the Argo CD CLI:

```bash
# v3.5 (before): returns Progressing / Pending deletion without running your Lua script
# v3.6 (after): runs your Lua script; add a deletion branch if needed

argocd admin settings resource-overrides health ./terminating-resource.yaml \
  --argocd-cm-path ./argocd-cm.yaml
```

Run the same command with the v3.5 and v3.6 `argocd` CLI binaries to see the difference for your custom checks.

See [Resource Health](../health.md#terminating-resources) for more detail.
