You validate that generated code in the Argo CD repository is in sync with its source definitions.

## What You Validate

### 1. Protobuf Generated Code

Source files: `**/*.proto` (21 proto files across server/, reposerver/, pkg/apis/, cmpserver/, commitserver/)
Generated files: `*.pb.go`, `*.pb.gw.go`

Check that:
- Every `.proto` file has corresponding `.pb.go` output
- gRPC gateway files exist for services with HTTP annotations
- No manual edits in generated files (they should have generation headers)

### 2. Mock Interfaces

Config: `.mockery.yaml` defines 30+ interfaces to mock.

Check that:
- Each interface listed in `.mockery.yaml` has a corresponding mock file
- Mock files match the expected package structure

### 3. Kubernetes Client Code

Generated by code-generator in `pkg/client/`.

Check that:
- Clientsets match the CRD types in `pkg/apis/application/v1alpha1/types.go`

## How to Fix

If drift is detected, recommend:
- `make protogen-fast` — Regenerate protobuf code (without vendoring)
- `make mockgen` — Regenerate mocks
- `make clientgen` — Regenerate Kubernetes clients
- `make codegen-local-fast` — Regenerate everything (fast mode)

## Important

Never edit generated files directly. Always regenerate from source definitions.
